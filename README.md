# GitHelp

GitHelp is a helpful AI agent designed to assist users with various Git and GitHub tasks. Whether you need help cloning a repository, pushing changes, fixing merge conflicts, or exploring GitHub repositories, GitHelp provides an interactive interface to execute these tasks efficiently.

## Features

- **Local File Management**: Read and write files directly in your local environment.
- **GitHub Integration**: List repositories, view repository structures, and read files directly from GitHub.
- **Command Execution**: Run bash commands safely with user confirmation.
- **Context-Aware Assistance**: Explores your project structure to provide relevant advice and automation.
- **Interactive AI**: Powered by an AI engine that understands natural language requests and translates them into actionable commands.

## Project Structure

- `main.py`: The entry point of the application, handling the main execution loop.
- `init.py`: Handles initialization and setup for the AI agent.
- `ai_to_commands.py`: The logic engine that translates AI responses into executable system commands.
- `prompt.txt`: The system prompt defining the AI's behavior and operating procedures.
- `.venv/`: Python virtual environment containing necessary dependencies (e.g., `PyGithub`, `google-genai`).

## Getting Started

### Prerequisites

- Python 3.14 or higher.
- A GitHub personal access token (stored in `auth.dat`).
- API credentials for the AI backend (stored in `api.dat`).

### Installation

1. Clone the repository:
   ```bash
   git clone <repository-url>
   cd githelp
   ```

2. Activate the virtual environment:
   ```bash
   source .venv/bin/activate
   ```

3. Run the application:
   ```bash
   python main.py
   ```

## Operating Procedures

GitHelp follows a strict protocol to ensure safety and clarity:
1. **Analyze**: Understands your request.
2. **Explore**: Gathers context from your files or GitHub.
3. **Propose**: Explains the intended actions before execution.
4. **Execute**: Performs the commands upon your confirmation.

## Safety

GitHelp will never perform destructive actions (like deleting repositories or overwriting critical system files) without explicit user confirmation for each command.
